# FEATURE.md — `transcript-studio`

**Status:** `scaffolded`
**Tier:** `1`
**Last updated:** `2026-06-17`

---

## Purpose

A 4-column live transcription workspace. Users record audio, see raw transcript stream into Column 1, watch an AI agent clean it in Column 2 every ~30s, see concepts extracted by a different agent in Column 3 every ~200s, and configure Column 4 with a pluggable module (default: action-item tasks; alternates: flashcards, decisions, quiz). Built for long sessions (1–3h meetings, lectures) with crash-safe IndexedDB safeguarding.

## Entry points

**Routes**
- `app/(a)/transcripts/studio/` — full-page workspace, sidebar + active session view. Public URL: `/transcripts/studio` (see `next.config.js` redirects from `/transcript-studio`).
- `app/(a)/transcripts/scribe/` — Scribe: voice-first capture + assistant + working-document workspace. Public URL: `/transcripts/scribe` (legacy `/transcripts/mobile` 308-redirects here). Reuses the same session + segment data layer; SSR-seeds via `StudioHydrator`, mounts `ScribeScreen` per session.

**Scribe capture + assistant** (`components/scribe/`)
- `ScribeScreen.tsx` — controller with a 3-mode header toggle: **Record** (`capture`) / **Agent** (`agent`) / **Live** (`live`). Single-row header (title hidden under `sm`), inline session-global recording control (tap to start, pulses red + stops while recording), and the ⋮ session menu. Activates the session (wires realtime) + loads recording/raw/cleaned segments + documents on mount. A just-finished recording fires a **toast** ("Send it to the agent…?" → `send(REVIEW_MESSAGE)`) — no dedicated strip/row.
- `ScribeCaptureScreen.tsx` — big record button + audio-reactive ring, pause/resume/stop, live transcript strip, and the recording-card list. Each re-tap of record after stop opens a new cycle.
- `RecordingCardList.tsx` / `RecordingCard.tsx` — one card per recording cycle (`studio_recording_segments`). Multi-select (checkbox + long-press) → bottom Keep/Delete bar (`confirm` + `deleteRecordingSegmentThunk`). Tap/long-press → full transcript drawer. Per-card audio playback via `useFileSrc(audio_path)` → HTML5 `<audio>`.
- `FullTranscriptDrawer.tsx` — Drawer showing one cycle's raw transcript with a `ContentActionBar`.
- `SessionAudioPlayer.tsx` — the ONE transport surface for a session's audio. Mounted once at `ScribeScreen` (above all tabs) so it owns the single `HTMLAudioElement` via `useScribeSessionAudio`; plays the whole session on its seconds-from-start timeline (auto-advancing across per-recording files) with scrub / ±10s / playback-speed. Surfaces only once engaged (a card tap or an agent `<audiocite>` seek). Recording cards no longer carry their own `<audio>` — they fire `requestScribeAudioSeek` at this player, which is what finally gave scribe playback real seek/rewind/ff.
- `AssistantScreen.tsx` (the **Agent** tab) — working document as a **collapsible accordion** (collapsed by default; the accordion header carries the one unavoidable separator) with "Read aloud" (`useCartesiaSpeaker`) + full-screen `FocusedDocumentEditor`. Below it, the conversation + input are the shared **`AgentConversationColumn`** (the same seamless transcript+composer used by `/chat` and the agent-run route) — no background shift, no divider, content scrolls under the input.
- `ScribeLiveScreen.tsx` (the **Live** tab) — realtime two-way **voice** chat with xAI Grok, reusing the `features/voice-agent/` primitives (hooks + inner components). Mounts a `playground`-preset instance (`persist: false`) and injects the session's working document into the agent instructions, refreshed via `updateConfig` whenever the document changes (read at each `session.update`). **Phase 1** = context injection only; **Phase 2** (later) would add xAI tool/function-calling for explicit doc read/write.

**Hooks**
- `hooks/useStudioAssistant.ts` — ensures the working document and resolves the session's **durable** assistant conversation via `ensureAssistantConversationThunk`, then rebuilds the named context objects (`recording_NN_raw` / `recording_NN_clean` / `all_raw` / `session_cleaned` / `working_document`) before each turn. Mounted at the `ScribeScreen` level (not just the Agent tab) so a clean that finishes on the capture screen still refreshes assistant context — and the **Live** tab (`ScribeLiveScreen`) shares the same hook, so both AI surfaces get identical context.
- `redux/ensureAssistantConversation.thunk.ts` — guarantees ONE persistent assistant conversation per session that survives refreshes. Reuses the id in the slice → else the id persisted on `studio_sessions.assistant_conversation_id` (fetching the row if not yet in Redux for direct navigation), creating the instance keyed by it and `loadConversation`-ing full history → else mints a fresh conversation and writes its id back to the session row. De-duplicated by `sessionId` (module-level in-flight map) so the hook's two mounts can't create two conversations.
- `service/assistantContextBuilder.ts` — builds the `setContextEntries` payload: per-recording `recording_NN_raw` + `recording_NN_clean` (the recording-aligned clean), the `all_raw` aggregate, the full-session `session_cleaned` (ordered concatenation of every `recording_NN_clean`), and the mutable+persisted `working_document` rich context object (`source.kind = "studio_document"`). (The old `cleaned_transcripts`/`scribe_cleanup` duplicate entry was removed — `session_cleaned` is now the single clean version.)
- `components/scribe/SessionTranscriptViewer.tsx` — ≥50%-height drawer showing a session's full transcript in one of two modes: **raw** (`all_raw`) or **clean** (`session_cleaned`). Copy via `ContentActionBar`; clean mode offers a "Refresh" that re-cleans every recording (`cleanRecordingThunk` per recording). Opened from the capture record bar (the **All raw** / **All clean** icons flanking the mic) and from the session ⋮ menu ("View raw / View clean transcripts"). Replaced the old one-shot `CleanupSheet`.
- `redux/cleanRecording.thunk.ts` — **recording-aligned cleaning**. On a recording's finalize (and on the manual per-recording / per-session refresh) it cleans exactly that recording's raw into ONE `studio_cleaned_segments` row anchored to `recording_segment_id`; supersession is scoped to the same recording so re-cleans are idempotent. `prior_cleaned_suffix` is built from earlier recordings' cleans for cross-recording continuity. Replaced Scribe's old time-windowed `runCleaningPassThunk`-on-stop.

## Data model

**Database tables** (Supabase)
- `studio_sessions` — parent. Multi-scope (`user_id`, `organization_id`, `project_id`, `is_public`). `transcript_id` is a nullable FK to `transcripts(id)` for bidirectional conversion with the simpler transcripts feature. `source` (`'studio'` default | `'cleanup'`) records the origin surface: the high-volume `/transcripts/cleanup` page creates sessions on this same schema. **Session lists are source-scoped by default** — `listSessions`/`listSessionsServer` take a `SessionListFilter`; omitted ⇒ `source <> 'cleanup'` (studio view), `{source:'cleanup'}` ⇒ cleanup view, `{source:'all'}` ⇒ everything. A cleanup session is a real studio session (raw → `studio_raw_segments`, clean → `studio_cleaned_segments`, custom output → `studio_documents` `kind='cleanup_custom'`) — opening one in the studio works; it's just hidden by default.
- `studio_recording_segments` — one row per start/stop cycle. `audio_path` holds the durable `cld_files` fileId for the cycle's assembled audio (set by `finalizeRecordingSegmentThunk`); raw chunks link via `studio_raw_segments.recording_segment_id`. Backs the mobile recording cards.
- `studio_documents` — the assistant's collaborative working document (migration `migrations/studio_documents.sql`). Edited server-side via `ctx_patch` (aidream writeback handler `kind="studio_document"`); structurally separate from `studio_cleaned_segments` so the auto-cleanup version is never overwritten. One row per `(session_id, kind)`.
- `studio_raw_segments` — append-only chunk log (Column 1).
- `studio_cleaned_segments` — versioned cleaned text (Column 2). `superseded_at` flips when a later run replaces from the same time anchor forward.
- `studio_concept_items` — Column 3 output (themes, key ideas, entities, questions).
- `studio_module_segments` — Column 4 output (polymorphic `payload jsonb` keyed by `module_id` + `block_type`).
- `studio_runs` — agent invocation audit trail (column 2/3/4, conversation_id, status).
- `studio_session_settings` — 1:1 with session. Per-session shortcut + interval + module overrides; bounds enforced via DB CHECK constraints (`cleaning_interval_ms BETWEEN 15000 AND 120000`, `concept_interval_ms BETWEEN 60000 AND 600000`, `module_interval_ms BETWEEN 15000 AND 1800000`). `context_items jsonb` holds per-session user-authored context items (`SessionContextItem[]`: `{id,key,label,value,noteId?}`) passed to agents as proper context entries — added for the cleanup page, available to the studio too (candidate for a settings drawer).

RLS uses the canonical `check_resource_access(...)` pattern. Child tables inherit access via `EXISTS` on the parent `studio_sessions` row.

## Key flows

### Phase 1 (current): list + create session
1. Route SSR-fetches `studio_sessions` via `listSessionsServer`, hands them to `StudioHydrator` which seeds Redux during the first render pass.
2. `StudioView` reads `fetchStatus` and runs a client-side `fetchSessionsThunk()` if SSR didn't seed.
3. `StudioSidebar` renders the list. "New" calls `createSessionThunk({ userId, activate: true })` → inserts a `studio_sessions` row → updates Redux → activates the new session.
4. The active session renders `ActiveSessionPlaceholder` until Phase 3 wires the 4-column workspace.

### Bidirectional conversion (Phase 9)
- "Promote to Studio" on any `transcripts` row → creates a `studio_sessions` row with `transcript_id` set + migrates `transcripts.segments` JSONB into `studio_raw_segments` rows.
- "Save as Transcript" on a studio session → materializes a `transcripts` row and back-links via `transcript_id`.

### Resume-marker cleaning (Phase 5 — Studio, time-windowed)
- The Studio's interval cleaner (`runCleaningPassThunk`) embeds the last ~1000 chars of prior cleaned output ending in `[[RESUME]]`, followed by the new raw text since that anchor. The agent replies starting with `[[RESUME]]`; we strip it and replace any `studio_cleaned_segments` rows whose `t_start >= replaceFromTime`. These rows have `recording_segment_id = NULL`.

### Recording-aligned cleaning (Scribe)
- A recording is the unit. When a recording closes, `cleanRecordingThunk` cleans **that recording's** raw → ONE `studio_cleaned_segments` row anchored to its `recording_segment_id`. The full-session clean (`session_cleaned`) is the ordered concatenation of those rows; there is no separate monolithic clean document. `prior_cleaned_suffix` carries earlier recordings' cleans for continuity. Supersession is scoped to the same `recording_segment_id` (`applyCleanupRun({ recordingSegmentId })`) so the per-recording / per-session **refresh** is idempotent.
- **Planned (next):** continuous capture with periodic auto-rotation — a long recording auto-splits into bounded (~N-minute) recording segments without the user tapping stop/start; each rotated segment cleans on close. The data model + cleaning + UI + context already support this (every segment is recording-aligned); only the recorder-level seamless rotation in `useChunkedRecordAndTranscribe` / `GlobalRecordingProvider` remains.

### Synchronized scrolling (Phase 4)
- Each segment renders a `<div data-tstart=N data-tend=M>` wrapper. A `wheel` / `touchstart` listener flips a column to "leader" for 600ms; while leader, an IntersectionObserver writes `cursorTime` to Redux. The other three columns binary-search their segments for `[t_start, t_end]` containing `cursorTime` and `scrollIntoView`. Per-column "autoscroll" flag pauses when the user scrolls past 80px from the bottom.

## Coexistence with `features/transcripts/`

These are sibling features. The simple `features/transcripts/` view is shaped for one-shot, finished transcripts (one JSONB segments blob per row). The studio uses per-segment rows because live append + per-segment realtime + indexed time queries don't fit a hot JSONB blob. Bidirectional one-click conversion is the integration contract — see `service/transcriptBridge.ts` (Phase 9). The route→store contract for the whole `/transcripts` ecosystem (two record stores + `cld_files` audio, bridge-only conversion, no third store) lives in **`features/transcripts/FEATURE.md`** — read it before adding any transcription surface.

## Invariants

- Every segment in every column has `t_start`/`t_end` in seconds-from-session-start, paused time excluded. This is the single coordinate system for sync scrolling.
- `studio_raw_segments` is append-only from the application's perspective (RLS allows updates for editor scope, but the recorder must never patch existing rows).
- `studio_cleaned_segments` keeps superseded rows for audit; the active selector filters `superseded_at IS NULL`.
- Module switch mid-session preserves prior module segments tagged with their original `module_id`. The active selector returns segments where `module_id === session.module_id`. A "show prior modules" toggle on the column header (`studio_session_settings.show_prior_modules`) reveals the rest.
- Recording lives in a global provider (Phase 2), not in `StudioView` — recording continues across route navigations.
- All agent runs go through `useShortcutTrigger` / `launchAgentExecution`; never call agents directly.
