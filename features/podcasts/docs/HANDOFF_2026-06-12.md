## Architecture cheat-sheet

**Host-count routing** (`packages/matrx-ai/matrx_ai/agent_runners/podcast_generator.py`):
```
1    -> podcast_solo_script        -> Gemini single voice (tts_voice:"<name>")
2    -> educational/news/persian   -> Gemini multi-speaker (tts_voice:[{name,voice}])
3-4  -> podcast_multihost_script   -> ElevenLabs dialogue (tts_voice:[{text,voice_id}])
5-20 -> podcast_roundtable_script  -> ElevenLabs dialogue
```
- `_select_script_agent` / `_create_audio` / `_dialogue_to_elevenlabs_turns`
  (raises `PodcastConfigError` BEFORE the paid TTS call if a script speaker has
  no voice mapping — loud failure, never a silent drop).
- `_effective_speakers` resolves names from `request.speakers` or the script's
  distinct labels, fills missing voices from `_GEMINI_DEFAULT_VOICES` /
  `_ELEVENLABS_DEFAULT_VOICES`.

**Streaming audio wire format** (Gemini only):
- `audio_stream_chunk` events: `{stream_id, seq, audio_base64 (s16le PCM),
  sample_rate:24000, bits_per_sample:16, channels:1}` then `audio_stream_end`.
- Defined `packages/matrx-connect/.../context/data_types.py`; emitted by
  `providers/google/google_api.py::_emit_audio_stream_chunk`.
- **ElevenLabs does NOT stream chunks** — `text_to_dialogue.stream` is collected
  server-side (`b"".join(...)`) and returns MP3; the client waits for the final
  URL. (See "Top follow-ups" to add live ElevenLabs streaming.)

**Client streaming player** (`features/audio/streamingPcmPlayer.ts`): generic
buffered PCM player (play/pause/seek/position). Consumed in
`features/podcasts/studio/runs/useStudioRun.ts` (folds `audio_stream_chunk` /
`audio_stream_end`, seq-gap detection → falls back to wait-for-URL) and shown via
`features/podcasts/generator/components/LiveAudioPlayer.tsx`; `StudioRunView`
swaps to `PodcastAudioPlayer` (now with `initialTime`/`autoPlay` handoff props)
when the canonical URL lands.

**Blog / show notes** (`features/podcasts/`):
- `articleService.ts` (CRUD on `pc_articles`), `generator/useEpisodeArticles.ts`
  (runs agents via `useRunAgent`, upserts), `studio/components/EpisodeContentStudio.tsx`
  (generate/stream/publish UI on the run page), public `app/(core)/podcast/[slug]/blog/`
  + `components/player/PodcastBlogPage.tsx`, `EpisodeShowNotes` on the episode page.
- Shared `features/podcasts/utils.ts#slugify` (extracted from 3 dupes this round).
