## Chunk findings

**F1 — STALE `deprecated` flag on two SHIPPED windows (P0·S, doc-correctness bug).** `agentFindUsagesWindow` and `agentAdminFindUsagesWindow` are real, live implementations (the find-usages + drift engine, `<AgentUsagesEngine>`, RPCs `agx_usage_scan` / `agx_usage_scan_admin`, fired from `AgentOptionsMenu` + drift DM chips). But `registry/windowRegistryMetadata.ts` (lines ~1032 and ~1062) still carries `deprecated: { note: "Stub — not yet implemented…" }` left over from when they were `AgentPlaceholderWindows` coming-soon stubs. Per `windowRegistryTypes.ts`, that flag makes `WindowPanel` paint a **red ring + "deprecated" banner** and the Tools-Grid tile render in **destructive red with a `*` prefix** — i.e. two production features are visibly marked deprecated to users. Just delete those two `deprecated` blocks.

**F2 — Tools-Grid hygiene: "(new)" tiles pointing at empty stubs + a demo gallery parked in `admin`.**
- Three "(new)" tiles in the **agents** category open empty `AgentComingSoonContent` placeholders: `tile.agent-optimizer` → `agentOptimizerWindow`, `tile.agent-interface-variations` → `agentInterfaceVariationsWindow`, `tile.agent-data-storage` → `agentDataStorageWindow`. Each is also a first-class item in `AgentOptionsMenu`. These present coming-soon screens as shipped capabilities.
- The **interface-variation DEMOS** that `agentInterfaceVariationsWindow` was meant to gallery are live overlays parked as ~10 first-class "(new)" tiles in the **admin** category: `agentFullModal`, `agentCompactModal`, `agentChatBubble`, `agentInlineOverlay`, `agentSidebarOverlay`, `agentFlexiblePanel`, `agentPanelOverlay`, `agentToastOverlay`, `agentFloatingChat`, `agentChatCollapsible` (`toolsGridTiles.ts` ~758–832). They are demos, not admin tools — they should live behind the variations gallery (or a dedicated demos surface), not as peers of real admin inspectors.

**F3 — `chatDebugWindow` has NO Tools-Grid tile.** It is the only debug window in this chunk missing from the grid; it is reachable only from chat input menus (`ConversationInput` ×2, `RunControlsMenu`). It is also misfiled into the "debug window" family — it is a **settings/toggle panel** (flips global/session debug, block mode, system-messages), not a state inspector, so it must NOT be folded into the consolidated inspector (F4). Fix: add an admin-gated `tile.chat-debug`.

**F4 — Inspector consolidation: one mode-driven inspector, not five windows (P1·L).** `agentDebugWindow` already is a 7-tab inspector over agent-definition + execution-instance Redux state. Four standalone windows are lenses on the SAME execution data at different scopes: `executionInspectorWindow` (instance inspector — overlaps `instances`/`uistate`), `streamDebugHistoryWindow` (stream replay — overlaps `history`), `messageAnalysisWindow` (per-message request stats — overlaps `history`/`uistate`), and the sibling `instanceUIStateWindow` / `adminStateAnalyzerWindow` / `streamDebug` widget (Admin chunk). Verdict: **merge-to-modes** — make agentDebug the canonical inspector head and absorb these as tabs/modes (definition · instances · variables · stream · request/message · ui-state · history), driven by `{agentId, conversationId, requestId, mode}` seed data. Keep `chatDebugWindow` OUT (it writes settings, doesn't read state). This collapses ~5 admin windows + 1 widget into one.

**F5 — Stubs: recommend DELETE for two, DECIDE for one.** `agentOptimizerWindow` and `agentDataStorageWindow` are empty coming-soon shells with no implementation, no roadmap evidence, surfaced as first-class menu items + tiles — run `remove-window-panel` on each (window + opener + Controller + catalogue + OverlayId + registry entry + Tools-Grid tile + `AgentOptionsMenu` item). `agentInterfaceVariationsWindow` is the one worth building rather than deleting: the demos it would gallery already exist (F2), so building it is mostly wiring an index over them — otherwise delete it and demote the parked demos.

**Orphan / reachability:** none in this chunk — every panel has an opener + Controller + at least the Tools-Grid tile (except `chatDebugWindow`, which has chat-input call sites but no tile, F3). `agentDebugWindow`, `streamDebugHistoryWindow`, `agentAssistantMarkdownDebugWindow`, `executionInspectorWindow` are Tools-Grid-only (0 bespoke call sites) — acceptable for admin diagnostics, but agentDebug would benefit from an AgentOptionsMenu entry post-consolidation.

**Maturity honesty:** of 11 panels — 6 Solid live inspectors (agentDebug, FindUsages×2, messageAnalysis, executionInspector, streamDebugHistory), 1 Solid settings panel (chatDebug), 1 niche Stub→Partial probe (mdDebug), 3 empty Stubs (optimizer, interfaceVariations, dataStorage). No Gold in this chunk (debug tools, by design). None is a registered surface and none wires help/agents/std-controls — correctly low priority for diagnostics.
